Aufbau eines Namens
Norm _ Stelle __ Locator _N<n>
Beispiel IEC62443_3_3_SR3_3__P2_S1 =
IEC 62443-3-3, Anforderung SR 3.3, Absatz 2, Satz 1.
Anforderung (Control)
Eine einzelne, nummerierte Anforderung aus dem Register der Norm — z. B. SR 3.3, CR 1.1, PRE-1. Du wählst sie in Schritt 2 aus einer durchsuchbaren Liste; die korrekte Kurz-ID setzt das Tool selbst ein.
Locator: von grob nach fein
Absatz → Satz → Buchstabe → Nummer. Immer nur so fein wie nötig; leer = ganze Anforderung.
Sonderfälle
Noch keine passende Anforderung: TODO_NEU
Ohne Normbezug, eigenes Wissen: INTERNES_WISSEN
Zweite Vergabe desselben Namens: _N2, _N3, …
Regeln von Word
Nur Buchstaben, Ziffern, Unterstriche · muss mit Buchstabe beginnen · max. 40 Zeichen. Das Tool hält sie automatisch ein.
Für Technische: formale Grammatik (EBNF) 
Textmarke     ::= Standardform | Sonderfall
Standardform  ::= Normkuerzel "_" Normstelle [ "__" Locator ] [ Kollision ]
Sonderfall    ::= ( "TODO_NEU" | "INTERNES_WISSEN" ) [ Kollision ]

Normkuerzel   ::= "IEC62443_2_1" | "IEC62443_3_3" | "IEC62443_4_1" | "IEC62443_4_2"
                | "EN40000_1" | "EN40000_2" | "EN40000_3" | "CRA" | "EN50742"

Normstelle    ::= ControlId | Kapitel
ControlId     ::= Buchstabe { Buchstabe | Ziffer | "_" }        (* Register-ID, z. B. SR3_3, CR1_1, SM_1 *)
Kapitel       ::= "K" Ziffer { Ziffer | "_" Ziffer }             (* z. B. K5_3 für Kapitel 5.3 *)

Locator       ::= [ Absatz ] [ "_" Satz ] [ "_" Buchstabenteil ] [ "_" Nummer ]   (* mind. 1 Teil, Reihenfolge fix *)
Absatz        ::= "P" Ziffer+
Satz          ::= "S" Ziffer+
Buchstabenteil::= "Lit" Buchstabe+
Nummer        ::= "Nr" Ziffer+

Kollision     ::= "_N" Ziffer+           (* _N2, _N3, … bei mehrfacher Vergabe desselben Namens *)

Buchstabe     ::= "A".."Z" | "a".."z"
Ziffer        ::= "0".."9"

(* Word-Textmarken-Regel: Gesamtname muss mit Buchstabe beginnen,
   nur [A-Za-z0-9_], maximal 40 Zeichen. *)
